Postman Collection Importer for Burp Suite is a powerful extension that seamlessly imports Postman collections and environments, converting them into Burp Repeater tabs and/or Sitemap entries. This allows for comprehensive security testing.

Features

Core Functionality:
  • Supports Postman Collection v2.0 & v2.1.
  • Allows environment file support with variable resolution.
  • Enables smart variable detection with guided resolution.
  • Supports multiple destinations: Repeater, Sitemap, or both.
  • Offers request preview and selective import.
  • Handles all authentication types: Bearer, Basic, API Key, OAuth2.
  • Supports all request body types: JSON, form-data, x-www-form-urlencoded.
  • Tracks progress and provides detailed logging.
Advanced Features:
  • Supports GraphQL: features automatic detection and operation extraction.
  • Smart Retry System: facilitates retrying failed requests with a single click.
  • Intelligent Variables: offers smart variable suggestions.
  • Multi-selection: allows choosing specific requests to import.
  • Rate Limiting: enables configurable delays for live requests (0-5000ms, default: 200ms).
  • Visual Indicators: provides color-coded status and variable resolution.

Usage

Quick Start: To perform a basic import, follow these steps:
  1. Load Collection: Click \Browse\ next to Collection and select your .json file.
  2. Load Environment (Optional): Select your Postman environment file.
  3. Choose Destination:
    • Repeater: For manual testing (no HTTP requests made)
    • Sitemap: For live requests and attack surface discovery
    • Both: To get the benefits of both.
  4. Configure Rate Limiting (Optional): Adjust delay between requests (default: 200ms).
  5. Import: Click \Import Collection\.
For variable resolution, follow steps 1-3 above. If unresolved variables are detected, choose a resolution method:
  • Upload Environment: Select additional environment file.
  • Manual Entry: Enter values with smart suggestions.
  • Ignore Variables: Proceed with unresolved variables.
  • Skip Problematic: Import only requests without issues.
Use Cases: Security Testing Workflows
  • API Penetration Testing: Import entire API collections for comprehensive testing.
  • Vulnerability Assessment: Convert Postman tests to Burp Scanner targets.
  • Manual Testing: Use Repeater tabs for detailed request manipulation.
  • Automation Bridge: Connect development tools with security testing.
Team Collaboration
  • DevSecOps Integration: Import from CI/CD Postman collections.
  • QA to Security: Reuse existing API test suites for security testing.
  • Documentation: Convert API documentation to live security tests.
